## Vecka 1 (29 juni–2 juli 2026)

### Måndag 29/6
- **Aktivitet**:
  - Diskuterade med **Claude** (AI-assistent) om **projektets struktur, funktionalitet och planering** för *FoundIt Fullstack Web*.
    - Analyserade **alternativa arkitekturer** (monorepo vs. separata repos, backend/frontend-integration).
    - Fastställde **tech stack**: Nuxt 3 + Vue 3 (frontend), Node.js + Express (backend), PostgreSQL + Prisma (databas), Better Auth (autentisering).
    - Definierade **MVP-omfång**: Sökfunktion, användarprofiler, watchlist, detaljsidor med SSR/SSG.
  - **Skapade ett tomt GitHub-repo** för projektet: [`foundit-fullstack`](https://github.com/JoseRaul-TR/foundit-fullstack/).
  - **Lagt till en MIT licens** [`Commit d4667b4
`](https://github.com/JoseRaul-TR/foundit-fullstack/commit/d4667b42688c9a6345944e30020d91db3819a22d).
  - **Skapade `LOGGBOK.md`** i repo:rot för att dokumentera utvecklingsprocessen veckovis.
  - **Skapade ett GitHub Project** med detaljerade *"Back Log Items"* och respektive *sub-items* enligt planeringen.[`FoundIt — Fullstack Development`](https://github.com/users/JoseRaul-TR/projects/4)

- **Beslut**:
  - Valde **monorepo med pnpm workspaces** för att underlätta beroendehantering och samarbete mellan frontend/backend.
  - Bestämde att **`@nuxtjs/i18n`** ska konfigureras från början (endast `en` aktivt i MVP) för framtida språksstöd (SV/ES).
  - Prioriterade **säkerhet från start**: Zod för validering, `express-rate-limit` för API-skydd, Helmet för HTTP-headers.

- **Problem**:
  - Osäkerhet om huruvida **GitHub Project** eller **repo-wiki** var bäst lämpat för *Back Log Items*.
  - Svårighet att strukturera *sub-items* på ett sätt som speglar beroenden mellan uppgifter.

- **Lösning**:
  - Använde **GitHub Project** (Kanban-style) för *Back Log Items*, då det ger en **visuell överblick** av uppgifter och deras status.
  - Delade in uppgifterna i **epics** (ex: "Backend", "Frontend", "Testing") och läncade *sub-items* till respektive *Issue*.

- **Reflektion**:
  - Att diskutera **arkitektur och tech stack** med en AI-assistent hjälpte till att **identifiera svagheter och möjligheter** i planeringen tidigt.
  - **Monorepo-structuren** kommer att förenkla beroendehantering och deployment, men kräver noggrann konfigurering av `pnpm workspaces`.

- **Resurser**:
  - [pnpm workspaces](https://pnpm.io/workspaces)
  - [GitHub Projects — Dokumentation](https://docs.github.com/en/issues/planning-and-tracking-with-projects/learning-about-projects/about-projects)
  - [Nuxt 3 — Monorepo Guide](https://nuxt.com/docs/guide/concepts/monorepo)

- **Lärdom**:
  - Att **strukturera projektet tidigt** (repo, loggbok, backlog) sparar tid och undviker kaos senare.
